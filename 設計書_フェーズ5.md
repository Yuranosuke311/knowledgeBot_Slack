# 設計書 フェーズ5：自動収集モード

## アーキテクチャ概要

```
[新規チャンネル参加]
  member_joined_channel
    → setAutoCollect(channelId)      # 自動収集を即時ON（確認なし）
    → postCollectConfirmation()      # フェーズ4：過去メッセージ確認（既存）

[親メッセージ投稿]
  message イベント（thread_ts なし or thread_ts == ts）
    → isAutoCollect(channelId) が true
    → processSingleMessage() で Notion ページ作成
    → notionPageId を Redis に保存（`notion_ts:{channelId}:{ts}`）

[スレッド返信投稿]
  message イベント（thread_ts あり、thread_ts != ts）
    → isAutoCollect(channelId) が true
    → Redis から親の notionPageId を取得
    → appendToKnowledgePage(pageId, replyText) で既存ページに追記

[一括スクリプト]
  scripts/enable-autocollect.ts
    → conversations.list で参加チャンネル一覧取得
    → 各チャンネルに setAutoCollect()
```

## Redis 設計

### 新規キー

| キー | 型 | TTL | 説明 |
|------|----|-----|------|
| `autocollect:{channelId}` | string (`"1"`) | なし | 自動収集モードON |
| `notion_ts:{channelId}:{ts}` | string (notionPageId) | 30日 | 親メッセージの Notion ページID。返信追記に使用 |

## 実装ファイル一覧

### 1. `lib/redis.ts`（追記）

```typescript
export async function isAutoCollect(channelId: string): Promise<boolean> {
  const val = await redis.get(`autocollect:${channelId}`)
  return val !== null
}

export async function setAutoCollect(channelId: string): Promise<void> {
  await redis.set(`autocollect:${channelId}`, '1')
}
```

```typescript
async function handleMessage(event: Record<string, unknown>): Promise<void> {
  if (event.bot_id) return
  if (event.subtype) return  // message_changed, message_deleted 等を除外
  // ※ スレッド返信（thread_ts !== ts）も収集対象とする

  const channelId = event.channel as string
  if (!await isAutoCollect(channelId)) return

  const ts = event.ts as string
  const rawText = ((event.text as string) ?? '').trim()
  if (rawText.length < 5) return

  const redisKey = buildRedisKey(channelId, ts, '_bulk')
  if (await isDuplicate(redisKey)) return

  // チャンネル名取得
  const slack = new WebClient(process.env.SLACK_BOT_TOKEN)
  let channelName = channelId
  try {
    const info = await slack.conversations.info({ channel: channelId })
    const ch = info.channel as Record<string, unknown> | undefined
    channelName = '#' + ((ch?.name as string) ?? channelId)
  } catch { /* 取得失敗時はIDをそのまま */ }

  await processSingleMessage(channelId, channelName, event as Record<string, any>)
}
```

**スレッド返信の扱い：**
- 返信は親メッセージとは独立した個別エントリとして Notion / Vector に保存
- `processSingleMessage` 内でタイトルは `rawText.slice(0, 80)` から生成されるため、返信であっても自然なタイトルが付く
- バルク収集（`collect/route.ts`）では親メッセージ取得時にスレッドを一括取得するため、リアルタイム収集との重複は Redis キーで防止済み

#### ④ Event Subscriptions に追加が必要なイベント

Slack App の **Event Subscriptions → Subscribe to bot events** に以下を追加：

| イベント | 用途 |

| `message.channels` | パブリックチャンネルの新規メッセージ受信 |
| `message.groups` | プライベートチャンネルの新規メッセージ受信 |

## 今後の拡張性（フェーズ6構想）

### タスク管理機能

フェーズ5で全メッセージを自動収集する基盤が整うことで、以下のタスク管理機能への拡張が可能になる。

#### 概要

投稿されたメッセージを Gemini で分析し、タスクの作成・完了を自動的に検出して Notion で管理する。

#### 動作イメージ

```
投稿: "来週月曜までにAさんの資料レビューをする"
→ Notion Tasks DB にタスク登録（期日: 来週月曜、内容: Aさんの資料レビュー）

投稿: "Aさんの資料レビュー完了しました"
→ Notion の該当タスクを「完了」に更新
```

#### 必要な実装

| 要素 | 内容 |
|------|------|
| Notion Tasks DB | タスク専用データベース（タイトル・担当者・期日・ステータス） |
| タスク検出 | Gemini でメッセージを分類（タスク作成 / 完了報告 / 該当なし） |
| 日付パース | 「来週月曜」「3/31」「今週中」などの自然言語日付を解釈 |
| タスクマッチング | 完了報告メッセージと既存タスクの対応付け（Gemini + Vector 検索） |

#### 技術的課題

- 完了報告と既存タスクのマッチング精度（AI判定のため100%ではない）
- 「来週月曜」などの相対日付の解析精度
- 誤検知（タスクでないメッセージをタスクと判定）への対処
