# 要件定義書 フェーズ5：自動収集モード

## 概要

リアクション不要で全メッセージを自動的に収集する「自動収集モード」を追加する。
ボットが参加しているチャンネルでは、すべての新規メッセージを自動で Notion + Vector に保存する。

---

## 背景・目的

現状（フェーズ1〜4）の収集トリガー：
- 過去メッセージ：ボット招待時に一括収集（フェーズ4）
- 新規メッセージ：📌💡✅ のリアクションをつけた場合のみ保存（フェーズ1）

課題：
- リアクションを忘れると保存されない
- 全メッセージを保存したいチャンネルでは手間がかかる

フェーズ5の目的：
- チャンネルに参加しているだけで、全メッセージを自動保存する
- リアクション機能（📌💡✅）を廃止する

---

## 機能要件

### FR-1：ボット参加時の自動収集有効化

ボットが新しいチャンネルに参加した際（`member_joined_channel`）：
- 確認メッセージなしで、**自動的に自動収集モードをONにする**
- フェーズ4の過去メッセージ収集確認（📚はいボタン）は引き続き表示する

### FR-2：新規メッセージの自動保存

自動収集モードが有効なチャンネルで新規メッセージが投稿された場合：
- リアクションの有無に関わらず Notion + Vector に保存する
- カテゴリは **「一般」** で保存

**収集対象と保存単位：**
- 親メッセージ → 新規 Notion ページを作成
- スレッド返信 → **親メッセージの Notion ページに追記**（1スレッド = 1ページ）

スレッドをひとつのページにまとめることで文脈が保たれ、RAG 検索の精度も向上する。

**スキップ条件：**
- `bot_id` あり（ボット投稿）
- テキストが5文字未満
- `subtype` あり（メッセージ編集・削除等のシステムイベント）

### FR-3：リアクション機能の廃止

フェーズ1で実装したリアクション（📌💡✅）による保存機能を削除する。

- `reaction_added` イベントハンドラを削除
- リアクション関連の型定義（`ReactionEmoji`・`CATEGORY_MAP`・`TARGET_REACTIONS`）を削除
- Slack App の Event Subscriptions から `reaction_added` を削除

### FR-4：既存チャンネルへの一括適用スクリプト

既にボットが参加しているチャンネルに対して、一括で自動収集モードをONにするスクリプトを提供する。

```bash
npx ts-node scripts/enable-autocollect.ts          # 全チャンネルにON
npx ts-node scripts/enable-autocollect.ts --dry-run # 対象チャンネルの確認のみ
```

---

## 非機能要件

- 自動収集の設定は Redis に永続保存（TTLなし）
- 重複防止は既存の Redis キー（`knowledge:{channelId}:{ts}:_bulk`）で対応
- リアクション機能は完全に削除し、自動収集のみでナレッジを蓄積する
